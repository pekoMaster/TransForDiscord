隱私政策

最後更新：2026-05-10

本隱私政策說明 Peko Embed（前稱 TFD / TransForDiscord，以下稱「本服務」）在 Discord 平台上運作時，如何收集、使用、儲存與保護您的資料。使用本服務即表示您同意本政策內容。

一、本服務簡介

Peko Embed 是一個 Discord 機器人，主要功能為偵測訊息中的特定 URL（如 Twitter/X、Pixiv、PTT、Threads 等），並產生增強的預覽 Embed。它需要 Discord 的 Message Content Intent 才能讀取訊息內容以偵測 URL。

二、我們收集的資料

2.1 自動處理但不儲存

本服務在偵測到支援的 URL 時會讀取以下資料以即時處理，處理完成後不留存：

  • 訊息文字內容（僅用於 URL 偵測與引用）
  • 訊息附件（僅用於重新發送至 Webhook）
  • 使用者顯示名稱與頭像（僅用於 Webhook 模擬發送）

2.2 短期儲存（伺服器端）

資料用途保留期限
URL 統計（規範化後的 URL key、出現次數、伺服器/頻道 ID）顯示「同一 URL 在本伺服器/頻道出現 N 次」於 footer7 天滾動 + 條目級 TTL
Rate-limit 計數（使用者 ID、伺服器 ID）防止單一使用者/伺服器濫用本服務5 分鐘滾動
濫用記錄（使用者 ID、頻道 ID、事件類型）偵測 spam 行為30 天
翻譯快取（譯文文字 hash）避免重複翻譯耗用 API 配額7 天

2.3 長期儲存

資料用途保留期限
您主動透過 /pe api add 提供的 AI API Key翻譯按鈕的 AI 翻譯來源直到您執行 /pe api del；AES-256-GCM 加密儲存
伺服器設定（log channel、排除清單）維持管理員設定的行為直到管理員修改或機器人離開伺服器

2.4 我們不會收集

  • 訊息歷史紀錄
  • 私訊內容
  • 您的真實姓名、Email、IP 位址
  • Discord OAuth Token
  • 不含支援 URL 的一般訊息內容

三、資料儲存與安全

3.1 加密

  • 使用者 API Key 在儲存前以 AES-256-GCM 加密
  • 加密金鑰僅存於主機端的環境變數
  • 即使資料庫檔案外洩，攻擊者無法解出 API Key

3.2 儲存位置

  • 所有資料儲存於服務主機的本地檔案系統（SQLite + JSON）
  • 不會傳送到任何第三方分析平台
  • 翻譯請求會經過您指定的 AI 服務商 — 這些請求受對應服務商的隱私政策約束

3.3 第三方服務

服務用途觸發時機
Discord API接收訊息、發送預覽全程
各支援平台（Twitter、Pixiv 等）抓取公開內容資料偵測到對應 URL 時
OpenAI / Anthropic / Google Gemini翻譯文字使用者點擊翻譯按鈕時

四、您的權利

4.1 查看

  • /pe api status — 查看您已設定哪些 AI provider
  • /pe status — 伺服器管理員可查看設定

4.2 刪除

  • /pe api del — 刪除您的 AI API Key
  • 伺服器管理員可隨時透過 /pe 指令修改設定
  • 移除機器人後，伺服器設定會在下一次 GC 清除

4.3 不參與

  • 伺服器管理員可使用 /pe nouser <你> add 將你排除於處理
  • 將 URL 用 <...> 包起來，Peko Embed 將不會處理

五、政策變更

本政策可能不定期更新。重大變更會在 GitHub repository 與機器人公告中公告至少 7 天。
