為什麼選擇 Peko Embed？

Discord 內建預覽經常壞掉、缺圖、不顯示影片。Peko Embed 幫你把每條連結都變成完美的預覽卡片。

🔗

網址預覽修正

自動偵測訊息中的 URL，產生增強的預覽 Embed。圖片、影片、文字一個不漏。告別 Discord 內建的破碎預覽。

🌐

一鍵推文翻譯

看到日文推文不用再切分頁！按一下翻譯按鈕，AI 幫你翻成中文。支援 DeepL 及多種 AI 翻譯引擎。

🖼️

多圖分頁瀏覽

Pixiv 多圖作品、PTT 多圖文章，不用點進去就能在 Discord 裡翻頁瀏覽。支援動圖(Ugoira)轉 MP4。

🎬

影片直接播放

Twitter 影片、Bilibili 影片直接在 Discord 內嵌播放，不用另開瀏覽器。支援 V2 Components 原生媒體展示。

🛡️

防爆雷系統

一鍵將預覽內容隱藏為防爆雷模式，保護還沒看到最新回的同好。適合追番、追劇、追 Vtuber 企劃的伺服器。

🚫

Per-Server 黑名單

各伺服器獨立設定平台作者黑名單。Level 1 提示、Level 2 自動防爆雷、Level 3 完全封鎖。管理員可透過指令或回報審核來管理。

🗑️

收回訊息

任何人皆可收回 Peko Embed 轉發的訊息。原作者可直接刪除，非原作者需填寫理由並記錄於日誌。每 10 分鐘上限 3 次防止濫用。

📋

回報系統

每則轉發訊息下方都有回報按鈕。可對內容上防爆雷、收回訊息、或回報作者至管理員審核黑名單。右鍵選單也支援同樣操作。

⚡

輕量無負擔

純 URL 預覽功能，不需要複雜設定。加入伺服器後自動運作，不干擾其他機器人，不儲存你的訊息內容。

準備好了嗎？

只需要一個按鈕，就能讓你的伺服器擁有完美的網址預覽。

＋ 將 Peko Embed 加入伺服器

📖 指令說明

所有設定都透過 /pe 斜線指令完成。加入伺服器後即自動運作，不需要任何設定。

所有人可用

/pe api add
provider: OpenAI | Claude | Gemini   apikey: 你的 Key
綁定你的 AI API Key，啟用推文翻譯功能。Key 以 AES-256-GCM 加密儲存，設定一次即跨伺服器可用。
/pe api edit
provider: OpenAI | Claude | Gemini   apikey: 新的 Key
修改已綁定的 API Key。
/pe api del
provider: OpenAI | Claude | Gemini
刪除已綁定的 API Key。刪除後該引擎的翻譯功能將停用。
/pe api status
查看你目前綁定了哪些 AI 服務商（不會顯示 Key 內容）。

管理員專用

/pe status
查看本伺服器的 Peko Embed 設定狀態，包含日誌頻道、排除清單、owner 等。
/pe log add / edit / del / show
channel: #頻道
設定、更換、移除或查看日誌頻道。開啟後，每次預覽轉換都會記錄到指定頻道。
/pe nouser
user: @使用者   action: 新增排除 | 移除排除 | 列表
排除特定使用者觸發預覽。被排除的使用者貼的連結不會被處理。
/pe noch
channel: #頻道   action: 新增排除 | 移除排除 | 列表
排除特定頻道觸發預覽。適合不希望出現預覽的頻道（如公告區）。
/pe blacklist add
platform: Twitter|PTT|Pixiv|YouTube|Instagram|Threads|其他   author: 作者名   level: 1|2|3   label: 標記（選填）
將平台作者加入本伺服器黑名單。Level 1=僅提示、2=自動防爆雷、3=完全封鎖不轉發。作者名支援 Twitter UID 防止改名規避。
/pe blacklist remove
platform: Twitter|PTT|...   author: 作者名
從本伺服器黑名單中移除指定作者。
/pe blacklist switch
action: on | off
啟用或停用本伺服器的黑名單系統。切換黑名單系統的啟用狀態。
/pe blacklist list
platform: 平台（選填）
列出本伺服器黑名單。可選平台過濾。每頁最多顯示 10 條記錄。
/pe owner
user: @使用者（選填）
設定本伺服器的 Peko Embed 活動 owner。不填則清除。
💡 小提示：不想讓某條連結被預覽？用 <https://...> 角括號包住 URL 即可跳過處理。

🛠️ 管理工具與操作說明

除了指令，Peko Embed 還提供按鈕互動與右鍵選單進行內容管理。

📋 回報按鈕

每則轉發訊息右下角都有 📋 回報 按鈕（60 秒內有效）。按下後出現三個選項：

  • 上防爆雷 — 套用防爆雷遮罩，隱藏預覽內容（可填理由）
  • 收回訊息 — 刪除轉發訊息。原作者可直接刪除；非原作者需填理由並記錄日誌
  • 黑名單回報 — 回報作者至管理員審核。選擇等級（1/2/3）並附理由

🖱️ 右鍵選單

對任何 Peko Embed 轉發的訊息 右鍵 → 應用程式 → PekoEmbed 操作，可直接執行：

  • 移除訊息 — 刪除轉發訊息（含日誌記錄）
  • 上防爆雷 — 對該訊息套用防爆雷
  • 黑名單回報 — 快速回報作者（需伺服器已設定日誌頻道）

🛡️ 黑名單執法機制

當使用者貼出連結時，Peko Embed 自動檢查作者是否在本伺服器黑名單中：

  • Level 1（提示） — 正常轉發，附加橘色警告標記
  • Level 2（防爆雷） — 自動套用防爆雷遮罩
  • Level 3（封鎖） — 不轉發內容，顯示提醒訊息

黑名單可透過 /pe blacklist add 手動加入，或經由使用者回報 → 管理員審核核准後自動加入。

👑 管理員審核流程

當使用者提交黑名單回報後，管理員會在日誌頻道收到審核訊息：

  1. 從下拉選單選擇最終等級（提示 / 防爆雷 / 封鎖）
  2. 點擊 ✅ 確認核准 並填寫審核理由
  3. 系統自動將作者加入黑名單（同作者重複加入會更新等級）

也可點擊 ❌ 拒絕 直接駁回回報。審核按鈕僅限擁有「管理伺服器」或「管理員」權限的成員使用。

💡 提示：管理員審核功能需要先設定日誌頻道（/pe log add）。未設定日誌頻道的伺服器，回報內容會顯示在當前頻道並附帶警告。
💡 注意：黑名單系統預設啟用。訊息上的回報按鈕（有效期 24 小時）與右鍵 PekoEmbed 操作才會生效。關閉時按鈕會顯示「本功能尚未啟用」。
